# Homework 1: Coding Assistant Onboarding

This repository contains my submission for Homework 1, which focused on using a coding assistant as part of a simple project workflow. The assignment required creating an AI-assisted draft, manually revising the content, and using GitHub to track progress through commits.

## Project Focus
For this assignment, I chose to write about a problem area I care about deeply: protecting vulnerable populations—especially aging individuals like my mother—from scams, spam calls, and social engineering attacks. I am interested in how technology, customer experience design, and analytics can be used to create stronger guardrails that help prevent people from being manipulated into giving away personal or financial information.

## Files Included
- `assistant_draft.md` – the initial draft created with the help of a coding assistant
- `final_version.md` – my manually revised version with more personal detail and clearer focus
- `README.md` – documentation of my workflow and submission details

## Coding Assistant Used
I used OpenAI and **Claude** as my coding assistant for this assignment.

## Git Steps I Used
1. Created a local project folder for the assignment
2. Initialized Git using `git init`
3. Created the required markdown files
4. Added and committed the initial repository setup
5. Used a coding assistant to generate the first draft
6. Copied the draft into a final version and manually edited it myself
7. Pushed the completed files to GitHub

## Commit Summary
- Initial repository setup
- Add assistant-created draft
- Revise final version with manual edits
- Update README

## Reflection
This exercise showed me that AI can be useful for creating a strong starting point, but it still needs human judgment to make the content more specific, meaningful, and accurate. I also learned more about using Git and GitHub as part of a real workflow rather than just writing in a document.
